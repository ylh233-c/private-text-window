# Private Text Window

Private Text Window 是一个 Windows 本地隐私草稿输入工具。它可以在小窗口中临时记录文本，并通过“最近字符显示”减少完整草稿在屏幕上的暴露。工具不依赖网络服务，完整内容仅保存在本地文件中。

它保留正常输入、换行、移动光标、删除、复制全部历史和展开编辑体验，同时默认只在主窗口中展示光标前指定数量的字符。

## AI 辅助生成说明

本项目代码主要由 AI 辅助生成，发布者已进行基础检查和本地运行验证。由于该工具会在本地保存输入历史，使用者在运行前仍应自行审查代码、启动脚本和数据保存逻辑，确认符合自己的安全与隐私要求。

## 隐私与安全边界

本工具只减少屏幕上的文本暴露，不提供加密保护。`history.txt` 为本地明文文件，请不要输入密码、Token、API Key、公司敏感信息、商业机密或其他高敏感内容。

本工具不包含联网、同步、上传或远程存储功能。所有历史内容和设置都保存在程序所在目录。

## 设计目标

- 减少完整草稿在屏幕上的暴露
- 保留正常输入、编辑、复制体验
- 不依赖网络和云服务
- 不引入第三方 Python 依赖
- 支持小窗口、置顶、无边框和透明度调节

## 功能特点

- 默认白色小窗口，可调透明度、字号和最近显示字数
- 默认启动时显示边框和设置栏，可一键切换无边框模式
- 隐藏模式下只显示光标前 N 个字符，不直接展示全部历史
- 展开模式下可像普通文本框一样编辑完整内容
- 支持中文、英文、空格、符号、粘贴和 Enter 换行
- 支持复制全部历史、二次确认清空历史
- 支持窗口置顶、任务栏图标、最小化到任务栏
- 历史和设置保存在本地，不依赖网络服务

## 适用场景

- 临时草稿
- 学习笔记片段
- 阅读、会议或资料整理时的短文本记录
- 需要局部显示的本地输入场景

## 不适用场景

- 密码管理
- Token / API Key 保存
- 公司敏感资料记录
- 需要真正加密保护的隐私数据

## 本地数据

完整内容保存在同目录的 `history.txt`，设置保存在 `settings.json`。这两个文件是本地个人数据，已在 `.gitignore` 中排除，不建议提交到 GitHub。

程序会自动生成并使用 `private_text_window.ico` 作为窗口和任务栏图标，该文件也已在 `.gitignore` 中排除。窗口打开期间会一直保留任务栏图标；即使切到无边框模式，也可以从任务栏找回窗口。

## 启动

推荐双击：

```text
run_private_text_window.vbs
```

`run_private_text_window.vbs` 仅用于无命令行黑框启动本地 Python 脚本，不包含联网、下载或远程执行逻辑。对 VBS 脚本不放心的用户可以改用 `run_private_text_window.bat`，或在命令行中运行：

```powershell
python .\private_text_window.py
```

备用入口：

```text
run_private_text_window.bat
```

每次启动都会默认显示边框和设置栏；透明度、字号、显示字数、窗口位置等设置会维持上一次状态。

## 环境要求

- Windows 10/11
- Python 3.10+
- 需包含标准库 `tkinter`
- 无第三方 Python 依赖

## 快捷键

- `Ctrl+B`：切换无边框 / 有标题栏和设置栏
- `Ctrl+E`：展开 / 收起全部内容
- 隐藏模式下 `Ctrl+C`：复制全部历史内容，保留换行
- 任意模式下 `Ctrl+Shift+C`：复制全部历史内容，保留换行
- `Ctrl+Shift+Delete`：二次确认后清空全部内容
- `Ctrl+Q`：退出
- `Ctrl+M`：最小化到任务栏
- `Ctrl+T`：切换置顶
- `Ctrl+Up` / `Ctrl+Down`：提高 / 降低窗口透明度
- `Ctrl+=` / `Ctrl+-`：增大 / 减小字体
- 隐藏模式下 `Ctrl+Right` / `Ctrl+Left`：增加 / 减少显示字数
- 隐藏模式下 `Left` / `Right`：在完整文本中移动输入光标

无边框模式下，拖动窗口顶部的白色细条可以移动窗口，拖动右下角的小区域可以缩放窗口。
